> 本轮不以“页面看起来能用”为完成标准。每项都需要同步检查组件契约、实现、样式、案例、测试、移动端和英文环境；重复出现的 Sticky、VirtualList 等问题必须增加可复现的回归测试。

## 第一批：案例基础设施与布局

### 1. Container / Flex / Grid 案例纯化

- [x] Container、Flex、Grid 的布局示意仅使用低圆角虚线框，不使用图片、渐变、阴影或装饰卡片。
- [x] `space-between` 等案例用多个虚线块直接表达排列、间距和对齐关系。
- [x] 案例在窄屏自动换行或收缩，不产生页面级横向溢出。

### 2. Space 组件

- [x] 新增 Space 组件并明确 `direction`、`size`、`alignment`、`wrap`、`fill`、`spacer` 等最小稳定契约。
- [x] 补齐类型、注册、样式、单测、文档案例、Props 表格和 `plan.md`。
- [x] 案例覆盖水平、垂直、自动换行、分隔内容和响应式场景。

### 3. Layout 视觉和场景扩充

- [x] 所有 Layout 案例统一使用低圆角虚线布局框，不混用实线和虚线视觉。
- [x] 增加 Baseline、Extended Toolbar、System Bar、Inbox、Constrained、Side Navigation、Three Column、Discord、Steam 等非单一后台场景。
- [x] 所有组合案例无意外内部滚动，并在手机端降级为合理的单列或抽屉式结构。

### 4. Playground Script 源码截断

- [x] 修复 Script 视图错误显示 `class="token ..."` 文本及丢失标签起始部分的问题。
- [x] 修复 Timeline 等数组/对象脚本被高亮 HTML 污染的问题。
- [x] 增加包含注释、字符串、对象数组、HTML 特殊字符的源码渲染回归测试。
