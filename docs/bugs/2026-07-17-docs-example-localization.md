# 文档案例中英文迁移计划

日期：2026-07-17

## 目标

- 文档 Header 切换 `LocaleProvider` 后，页面标题、案例标题、案例内容、交互状态、源码预览和 API 描述同步切换。
- 中文与英文共享同一套案例结构和状态，不复制两份组件实现。
- 每页集中维护文案映射，避免模板中散落语言判断。
- 案例使用真实业务卡片、面板和数据场景，避免用孤立按钮或色块充当布局示例。

## 实现约定

- 使用 `src/pages/docsLocale.ts` 的 `createDocsTranslator` 读取当前 `LocaleProvider`。
- 每个页面或案例在文件顶部集中声明 `zh` / `en` 文案；状态集中、派生数据集中、方法集中。
- 展示给用户的示例源码也必须使用当前语言，不能只翻译标题。
- API 表格描述纳入同一套翻译；属性名、事件名和类型定义保持代码原文。
- 每完成一个组件页面，至少验证中文、英文、窄屏布局、控制台错误、相关组件测试和生产构建。
