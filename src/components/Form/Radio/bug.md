# Radio 单选组件 Bug 记录与修改情况

| Bug 描述                         | 根本原因分析                                                                                                                                                                                                                                                                                                                                                        | 修复方案与改动细节                                                                                                                                                                                                                                                                                                                                 | 状态       |
| :------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------- |
| Radio 页面的所有案例都点击无效。 | **注入失效导致选择失效**：框架底座在 provide/inject 查找中依赖 local `Symbol`，但在多包加载 monorepo 环境下各 Symbol 不相等。这让 `Radio` 无法通过 `inject(RADIO_GROUP_KEY)` 获取到父 RadioGroup 上下文。因而，被包裹在 Group 中的单选框无法找到父级去调用 `changeEvent` 更新全局 modelValue，仅触发内部 update:modelValue 事件但又无父级去监听，导致点击毫无效果。 | 1. **底层修复**：将框架底层 `inject.ts` 中的 `PROVIDES_KEY`、`INSTANCE_KEY` 和 `createInjectionKey` 全部使用 `Symbol.for` 全局共享键名重构，实现多实例下父子链上下文的安全透传。<br>2. **阻断改造**：在 `Radio` 组件模板子节点 `.dot` 和 `.label` 上增设 `@click.stop.prevent="select"` 绝对拦截，结合逻辑层 `e?.stopPropagation()` 双重锁死冒泡。 | **已修复** |
