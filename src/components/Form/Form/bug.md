# Form 表单容器组件 Bug 记录与修改情况

| Bug 描述                             | 根本原因分析                                                                                                          | 修复方案与改动细节                                                                                                                                         | 状态       |
| :----------------------------------- | :-------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------- |
| 原生表单提交事件触发整页刷新         | `<form>` 标签在响应 `@submit` 事件时，若不进行额外拦截拦截，会走浏览器原生提交机制并刷新页面，打断 SPA 单页应用状态。 | 在 setup 中增加对原生 submit 事件的处理函数 `onSubmit`，执行 `e.preventDefault()` 完美阻断原生刷新动作，并向上派发框架的自定义 `submit` 事件。             | **已修复** |
| 表单项动态注销时偶现注册表未同步清理 | 之前仅在 unmounted 时使用 filter 过滤列表，若某些复杂切换时机下可能会遗漏注销。                                       | 改用闭包返回机制，由 `registerItem` 直接返回注销销毁闭包 `() => void`，并在 `unregisterItem` 时使用 `splice` 精确按照 index 擦除，消除内存泄露和状态错乱。 | **已修复** |
