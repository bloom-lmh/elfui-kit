# RadioGroup 单选框组组件 Bug 记录与修改情况

| Bug 描述                                     | 根本原因分析                                                                                                                           | 修复方案与改动细节                                                                                                           | 状态       |
| :------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------- | :--------- |
| 多打包实例下，子 Radio 无法获取 Group 上下文 | `RADIO_GROUP_KEY` 使用了本地的 `Symbol` 实例化，在不同打包 bundle 下会分配不同的 `Symbol` 值，导致 `inject` 无法查找到父级提供的状态。 | 升级为基于 `Symbol.for` 的全局共享 Symbols，保证不同运行时 bundle 下 Symbol 跨边界一致性。                                   | **已修复** |
| 初始值变化时没有同步更新内部 Ref 缓存        | 当 `modelValue` 属性被外部动态改变时，内部 Ref 缓存未做重置。                                                                          | 引入 `useEffect` 对 `props.modelValue` 进行依赖追踪，当外部 props 变动且与内部不一致时，使用 `.set` 精准更新内部响应式缓存。 | **已修复** |
